# RoomFormer 算法流程

## 1. 总体流程概览

本项目的核心流程可以概括为三段：

1. 数据预处理：将原始 3D 点云投影为俯视角单通道密度图，并将 3D 标注同步归一化到 2D 图像坐标系。
2. 模型输入与输出：模型读取密度图 `png` 作为输入，输出房间多边形角点、角点有效性，以及语义丰富版本中的房间类别/门窗类别。
3. 模型输出后处理：将原始网络输出筛选成有效房间、多边形和门窗线段，再进行评估与可视化保存。

下面按代码执行链路展开说明。

## 2. 数据预处理：点云 -> 图像

### 2.1 点云读取与准备

Structured3D 预处理入口是 [data_preprocess/stru3d/generate_coco_stru3d.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/stru3d/generate_coco_stru3d.py)。主流程在 `main()` 中：

- 第 77-80 行读取 `point_cloud.ply`
- 第 83 行调用 `generate_density(xyz, width=256, height=256)`
- 第 86 行调用 `normalize_annotations(scene_path, normalization_dict)`
- 第 106/112/118 行调用 `export_density(...)` 将图像保存到 `train/val/test`

其中，点云文件的读取由 [data_preprocess/common_utils.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/common_utils.py) 的 `read_scene_pc()` 完成，第 6-14 行将 `.ply` 解析为 `numpy` 数组。

如果需要从全景 RGB-D 原始数据先生成点云，则入口是 [data_preprocess/stru3d/generate_point_cloud_stru3d.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/stru3d/generate_point_cloud_stru3d.py)，它调用 [data_preprocess/stru3d/PointCloudReaderPanorama.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/stru3d/PointCloudReaderPanorama.py) 的 `generate_point_cloud()`，把多视角深度图重建为 `point_cloud.ply`。

### 2.2 点云投影为俯视密度图

真正的“点云 -> 图像”转换代码在 [data_preprocess/stru3d/stru3d_utils.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/stru3d/stru3d_utils.py) 的 `generate_density()` 中：

- 第 24-26 行对坐标轴做变换
- 第 30-35 行统计点云包围盒并外扩 10%
- 第 44-48 行把三维点的 `x,y` 坐标归一化到 `256x256` 图像平面
- 第 50-59 行统计每个像素位置的点数，生成密度图 `density`

这里生成的不是普通 RGB 图像，而是俯视角单通道 density map。像素值越大，表示该俯视位置累积到的点越多。

### 2.3 标注同步投影到 2D

为了让训练标签与输入图像落在同一坐标系，`generate_coco_stru3d.py` 在第 86 行调用 `normalize_annotations()`。该函数位于 [data_preprocess/stru3d/stru3d_utils.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/stru3d/stru3d_utils.py)：

- 第 67-81 行 `normalize_point()` 使用与密度图相同的归一化参数，将 3D 标注点映射到 2D 图像坐标
- 第 83-98 行分别处理 `lines` 和 `junctions`

随后：

- 第 100-142 行 `parse_floor_plan_polys()` 从结构化标注中恢复出房间、门、窗对应的多边形/线结构
- 第 172 行开始的 `generate_coco_dict()` 将这些标注转为 COCO 风格
- 第 196-209 行会把门窗从窄矩形进一步压缩为线段表示

### 2.4 图像保存

保存密度图的函数是 [data_preprocess/common_utils.py](/home/dzy/py_project/CAD/RoomFormer/data_preprocess/common_utils.py) 的 `export_density()`：

- 第 39-42 行将 `float32` 密度图乘以 255，转成 `uint8`
- 然后调用 `cv2.imwrite()` 保存为 `scene_id.png`

最终，模型真正读取的输入图像就是这些 `train/val/test/*.png`。

## 3. 模型的输入与输出

### 3.1 数据加载

模型的数据集入口在 [datasets/poly_data.py](/home/dzy/py_project/CAD/RoomFormer/datasets/poly_data.py)。

`MultiPoly.__getitem__()`：

- 第 49-50 行读取 COCO 标注
- 第 52-54 行在非语义模式下去掉 door/window
- 第 58 行调用 `ConvertToCocoDict`

`ConvertToCocoDict.__call__()`：

- 第 72 行读取预处理得到的 `png`
- 第 87 行或第 93 行将图像整理成形状为 `1 x H x W` 的张量
- 图像值会除以 255，变成 `[0,1]` 范围

因此，模型输入不是三通道彩色图，而是单通道俯视密度图。

### 3.2 模型结构入口

模型构建入口在 [models/__init__.py](/home/dzy/py_project/CAD/RoomFormer/models/__init__.py)，最终会调用 [models/roomformer.py](/home/dzy/py_project/CAD/RoomFormer/models/roomformer.py) 的 `build()` 和 `RoomFormer`。

在 `RoomFormer.__init__()` 中：

- 第 44 行 `class_embed` 负责角点有效性分类
- 第 45 行 `coords_embed` 负责角点坐标回归
- 第 101-102 行当 `semantic_classes > 0` 时，额外启用 `room_class_embed`，用于语义类别预测
- 第 105 行 `num_queries_per_poly = num_queries // num_polys`，说明模型是“每个 polygon 一组 corner queries”的两级查询结构

### 3.3 前向传播输出

`RoomFormer.forward()` 位于 [models/roomformer.py](/home/dzy/py_project/CAD/RoomFormer/models/roomformer.py) 第 116 行开始：

- 第 132 行：backbone 提取多尺度特征
- 第 157-160 行：将 query embedding、target embedding 和特征一起送入 transformer
- 第 163-164 行：将 transformer 输出 reshape 成“按房间组织”的形式
- 第 166 行：输出基础结果

基础输出有两项：

- `pred_logits`：每个 polygon 内各个 corner query 的有效性分数
- `pred_coords`：每个 corner query 预测的二维归一化坐标

语义丰富版本额外输出：

- 第 169-171 行：`pred_room_logits`

这里的 `pred_room_logits` 是直接由模型头 `room_class_embed` 产生的，不是额外后处理模块再推出来的。它表示每个 polygon 的语义类别，包括普通房间类别以及 door/window 类别。

## 4. 模型输出的后处理

### 4.1 评估入口

评估脚本 [tools/eval_stru3d_sem_rich.sh](/home/dzy/py_project/CAD/RoomFormer/tools/eval_stru3d_sem_rich.sh) 会调用 `eval.py`。在 [eval.py](/home/dzy/py_project/CAD/RoomFormer/eval.py) 中：

- 通过 `build_model(args, train=False)` 构建模型
- 通过 `build_dataset(...)` 读取测试集
- 第 139-145 行调用 `evaluate_floor(...)`

真正的后处理逻辑主要在 [engine.py](/home/dzy/py_project/CAD/RoomFormer/engine.py) 的 `evaluate_floor()` 中。

### 4.2 从原始张量恢复有效几何结果

在 [engine.py](/home/dzy/py_project/CAD/RoomFormer/engine.py)：

- 第 251-253 行读取模型输出 `pred_logits` 和 `pred_coords`
- 第 254 行用 `torch.sigmoid(pred_logits) > 0.5` 生成 `fg_mask`，筛掉无效角点
- 第 256-258 行若存在 `pred_room_logits`，则先做 `softmax`，再取最大概率类别 `pred_room_label`

随后按场景、按房间处理：

- 第 287-293 行：取出一个 polygon 对应的有效角点，并将归一化坐标乘 255 还原到图像坐标
- 第 297-298 行：普通模式下，仅保留顶点数不少于 4 且面积大于 100 的房间
- 第 301-308 行：语义丰富模式下，根据 `pred_room_label` 分流

具体规则是：

- 类别不是 `16/17` 时，视为普通房间，加入 `room_polys` 和 `room_types`
- 类别是 `16/17` 且角点数为 2 时，视为门窗线段，加入 `window_doors` 和 `window_doors_types`

因此，后处理的核心不是再做复杂几何优化，而是基于阈值、面积和类别规则，把模型输出整理成“房间多边形 + 门窗线段”。

### 4.3 评估与可视化

整理完成后：

- 第 315-319 行把 `room_polys`、`room_types`、`window_doors` 送入评估器 `evaluate_scene(...)`
- 第 332-348 行保存语义丰富预测图 `*_sem_rich_pred.png`
- 第 355-365 行保存轮廓叠加图 `*_pred_room_map.png`
- 第 237-248 行保存语义丰富 GT 图 `*_sem_rich_gt.png`

其中可视化函数在 [util/plot_utils.py](/home/dzy/py_project/CAD/RoomFormer/util/plot_utils.py)：

- `plot_semantic_rich_floorplan()`：绘制语义房间、门、窗
- `plot_room_map()`：将预测 polygon 轮廓叠加到 density map 上

## 5. 总结

这个项目的工程逻辑比较清晰：先把点云压成 2D 密度图，再让 Transformer 直接预测房间多边形及语义，最后通过轻量级规则把网络输出整理成可评估、可视化的结构化结果。换句话说，输入虽然来源于点云，但模型本身处理的是 2D 图像；语义类别也是模型直接输出的一部分，后处理主要负责筛选、组织和展示，而不是额外再做一次语义识别。
